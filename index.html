PERT Network Diagram (Activity-on-Node)

Detail:
📖 How to read this diagram (ES, EF, LS, LF, slack — every term defined)
ESDUREF
Activity name
owner • dates
LSSLACKLF
  • ES — Early Start: the soonest the activity can begin, given its predecessors (counted in working days/weeks from project start).
  • DUR — Expected duration: the PERT weighted average (O + 4M + P) ÷ 6.
  • EF — Early Finish: ES + DUR — the soonest it can end.
  • LS — Late Start: the latest it can begin without delaying the project finish.
  • SLACK (float): LS − ES — how much it can slip for free. 0 = critical path (red): any delay here moves the end date.
  • LF — Late Finish: the latest it can end without delaying the project.
  • Arrows run left→right from predecessor to successor. Plain arrows are finish→start; labelled ones (SS start→start, FF finish→finish, SF start→finish, +n/−n lag or lead) mark non-standard links. Red arrows trace the critical path.
  • Milestones are ◆ diamonds — zero-duration checkpoints (standard activity-on-node notation); they auto-complete when every predecessor reaches 100%. Bottom strip of each task box = % complete; ✓ green = done. Hover any cell for its live values; click any shape to edit.

PMI-style nodes: top row ES · DUR · EF, bottom row LS · SLACK · LF. Red = critical path, green = complete. Columns are dependency levels: everything in a column can only start after its predecessors to the left.

Gantt Chart — Schedule & Progress

⏱ Schedule reserves

Buffers added AFTER the estimated work, drawn as hatched bars at the end of the Gantt. Contingency = buffer for the known risks in your RAID log — it's inside the date you commit to the client (easiest: click from P80 and the committed date = the 80%-confidence Monte-Carlo finish). Management reserve = extra held by leadership for surprises, never promised. Quote the committed date externally.
